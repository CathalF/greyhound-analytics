## Decisions Made

### Architecture Decisions

**Shared rate limiter across all scrapers (global 2-sec limit)**
- Rationale: Single TokenBucketRateLimiter instance shared by all GreyhoundStatsScraper instances ensures global rate limiting across entire batch job
- Impact: Respects site's rate limits regardless of parallel scraping attempts

**Resume strategy: progress file tracks today's work**
- Rationale: Progress file (`.scraper_progress.json`) stores dog names, timestamps, and success/failure status for current day
- Impact: Batch job can resume after crashes without re-scraping already-processed dogs
- Implementation: Compares progress file date to current date - if same day, resumes; if different day, starts fresh

**Logging: INFO to console, DEBUG to file**
- Rationale: Console shows high-level progress (batch start/end, dog processing), file logs detailed debugging info
- Impact: Easy to monitor batch runs in real-time, detailed logs available for troubleshooting
- Implementation: RotatingFileHandler with 10MB max size, 5 backups

**Failure handling: continue on errors, log and move to next dog**
- Rationale: One dog's failure shouldn't crash entire batch
- Impact: Batch completes even if some dogs fail, failures logged for review
- Implementation: asyncio.gather with return_exceptions=True, try/except in scrape_dog()

### Data Access Strategy

**Top Ratings page as dog source**
- Discovered 31+ dog URLs from single listing page
- Focus on active, high-performing dogs most likely to appear in upcoming races
- Can be expanded to other listing pages (Top 50s, Graded Greyhounds) in future

## Issues Encountered

### Minor Issues (Resolved)

**Unicode encoding error with checkmark characters on Windows console**
- Issue: `\u2713` (✓) character caused UnicodeEncodeError in logging on Windows cmd
- Impact: Logging errors in console output (but logs still written to file)
- Resolution: Replaced Unicode characters with ASCII equivalents ([OK], [FAIL])

## Usage

### Running the Batch Scraper

```bash
# Run full batch (all dogs from Top Ratings page)
python -m src.batch_scraper

# Test with first 10 dogs
python -m src.batch_scraper --limit 10

# Force re-scrape (ignore progress file)
python -m src.batch_scraper --force

# Combination: force re-scrape first 5 dogs
python -m src.batch_scraper --limit 5 --force
```

### Monitoring

**Console output:**
- Real-time batch progress
- Dog-by-dog processing status
- Summary statistics at end

**Log file (`logs/batch_scraper.log`):**
- Detailed timestamps for all operations
- DEBUG-level information for troubleshooting
- Persistent record of batch runs
- Automatic rotation (10MB max, 5 backups)

**Progress file (`.scraper_progress.json`):**
- Tracks which dogs processed today
- Resume capability after crashes
- Success/failure status per dog

### Typical Daily Workflow

1. **Morning batch run:** `python -m src.batch_scraper` (updates all dog stats)
2. **Check logs:** Review `logs/batch_scraper.log` for any failures
3. **Resume if needed:** Re-run same command (skips already-processed dogs)
4. **Phase 3:** Scrape oddschecker for today's races and odds
