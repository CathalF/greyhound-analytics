<task type="checkpoint:decision" gate="blocking">
  <decision>Select dog name matching strategy</decision>
  <context>Oddschecker dog names may not exactly match greyhoundstats.co.uk names (capitalization, spacing, formatting differences). Need strategy for matching.</context>
  <options>
    <option id="exact-match">
      <name>Exact Match (Case-Insensitive)</name>
      <pros>Simple, fast, no ambiguity. Works if names are consistently formatted.</pros>
      <cons>May miss matches if sites use different formatting (e.g., "Proper Heiress" vs "PROPER HEIRESS" vs "Proper-Heiress")</cons>
    </option>
    <option id="fuzzy-match">
      <name>Fuzzy Matching (Levenshtein Distance)</name>
      <pros>Handles typos and minor differences. More robust to formatting variations.</pros>
      <cons>More complex, slower, risk of false matches (e.g., similar dog names)</cons>
    </option>
    <option id="normalize-match">
      <name>Normalized String Match (Recommended)</name>
      <pros>Remove spaces/hyphens, lowercase, then match. Handles most formatting differences without false positives. Fast and reliable.</pros>
      <cons>May still miss matches if names differ significantly (rare)</cons>
    </option>
  </options>
  <resume-signal>Select: exact-match, fuzzy-match, or normalize-match</resume-signal>
</task>
